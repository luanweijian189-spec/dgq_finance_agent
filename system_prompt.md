智能荐股投研助手系统设计方案
1. 系统概述 (Executive Summary)
本系统旨在解决内部投研团队面临的核心痛点：有效管理、追踪和评估来自专业金融人士的股票推荐信息。通过自动化信息抓取、构建动态股票池、建立多维度评估模型，本系统将零散的、易被遗忘的微信推荐信息，转化为结构化的、可量化评估的投研资产。最终目标是提升信息利用效率，识别出高质量的股票和可靠的信息源，从而实现投资收益最大化。系统的核心交互将围绕一个智能Agent/机器人展开，提供高效、便捷的人机交互体验。
2. 系统设计原则
* 自动化优先 (Automation First): 最大程度地减少人工干预，实现从信息获取、处理到评估的全流程自动化。
* 数据驱动 (Data-Driven): 所有的评估和决策都基于客观数据，包括市场行情数据和历史推荐表现。
* 用户友好 (User-Friendly): 通过Web界面和智能机器人，提供直观的数据展示和便捷的操作入口。
* 模块化与可扩展 (Modular & Extensible): 系统采用模块化设计，确保未来可以轻松接入新的信息源、评估算法或功能。
3. 系统架构 (System Architecture)
我们将系统设计为以下几个核心模块：
1. 数据采集层 (Data Ingestion Layer):
    * 微信消息抓取模块 (WeChat Connector): 负责实时/准实时地从指定的微信群或联系人处抓取消息。
2. 数据处理层 (Data Processing Layer):
    * 自然语言处理模块 (NLP Engine): 负责从原始消息中识别和提取关键信息。
3. 核心应用层 (Core Application Layer):
    * 股票池数据库 (Stock Pool Database): 系统的中央数据仓库，存储所有相关信息。
    * 评估与分析引擎 (Evaluation & Analysis Engine): 系统的“大脑”，负责每日执行评估任务。
    * 外部数据接口 (External Data API): 连接第三方金融数据服务。
4. 交互与展现层 (Interaction & Presentation Layer):
    * Web管理后台 (Web Dashboard): 提供可视化的数据展示、分析图表和人工操作界面。
    * 智能Agent/机器人 ( "OpenClaw" Agent): 用户与系统交互的主要入口，提供查询、操作和告警功能。

4. 模块详细设计
* 技术选型:
    * 方案A (推荐): Wechaty/UOS协议。 通过托管微信号的桌面版协议，模拟用户登录，稳定性较高，功能强大，可以获取历史消息和实时消息。需要一台专用的服务器或虚拟机7x24小时运行。
    * 方案B (备选): OCR + App自动化。 在安卓模拟器中运行微信，通过图像识别（OCR）和自动化脚本来读取消息。开发复杂，稳定性较差，但作为备用方案可以考虑。
* 核心功能:
    * 实时监听: 监听指定联系人/群聊的新消息，一旦有新消息立刻推送至数据处理层。
    * 历史同步: 提供一次性或定期的历史消息同步功能，确保不遗漏过去的重要推荐。
    * 消息去重与过滤: 过滤掉无关的日常聊天、图片、表情包等，只保留文本消息进行下一步处理。
* 核心功能:
    1. 意图识别 (Intent Recognition): 判断一条消息是否包含“股票推荐”意图。
        * 方法: 基于关键词（如“看好”、“关注”、“逻辑”、“代码”）、正则表达式（匹配股票代码格式 (60|00|30|68)\d{4}）和简单的机器学习分类模型。
    2. 实体提取 (Entity Extraction): 从被识别为“推荐”的消息中，提取结构化信息。
        * 股票代码/名称: [A股代码/名称] - 通过正则和股票词典精准提取。
        * 推荐人: [微信昵称/群内备注] - 从消息元数据中直接获取。
        * 推荐逻辑: [推荐的核心理由] - 这是最难的部分。初期可以提取包含关键词的整个句子；进阶可以利用文本摘要算法（如TextRank）或小型LLM模型来生成简短的核心逻辑摘要。
        * 推荐时间: [消息发送时间戳] - 从消息元数据中获取。
* 处理流程: 原始微信消息 -> 意图识别 -> (是推荐消息) -> 实体提取 -> 结构化JSON数据 -> 存入数据库
* 技术选型: PostgreSQL 或 MySQL。
* 核心数据表设计 (Schema):
    * recommenders (荐股人表):
        * id, name (昵称), wechat_id, reliability_score (可靠性评分), notes (人工备注)
    * stocks (股票基础信息表):
        * id, stock_code, stock_name, industry (行业)
    * recommendations (推荐记录表):
        * id, stock_id, recommender_id, recommend_ts (推荐时间), initial_price (推荐日收盘价), original_message (原始消息), extracted_logic (提取的逻辑), status (追踪中/已结束)
    * daily_performance (每日表现快照表):
        * id, recommendation_id, date, close_price, high_price, low_price, pnl_percent (相较于推荐价的涨跌幅), max_drawdown (最大回撤), evaluation_score (当日综合评分), notes (当日分析)
这是系统的核心价值所在。它会作为一个定时任务（如每天收盘后）运行。
* 两大评估模型:
    1. 股票质量评估 (Stock Quality Score - SQS): 针对单次推荐进行打分。
        * 盈利能力 (权重: 60%):
            * 当前收益率、峰值收益率、夏普比率（考虑波动风险）。这是最核心的指标。
        * 逻辑验证 (权重: 20%):
            * 系统自动从财经新闻源（如东方财富、财联社）抓取与该股票相关的新闻。
            * 通过关键词匹配，判断推荐逻辑（如“业绩超预期”、“中标大单”）是否在后续得到验证。初期可设为人工标记，后期可NLP自动判断。
        * 量化指标 (权重: 20%):
            * 市值范围: 是否处于适合博弈的区间。
            * 弹性 (波动率): 历史波动率，高弹性意味着潜在收益和风险都高。
            * 流动性: 日均成交额。
        * 输出: 每日给出一个0-100的动态评分，并附上一段文字分析，例如：“【XX股票-85分】由[张三]推荐，当前盈利15%。推荐逻辑‘中标’已于昨日公告证实。近期成交量放大，弹性较好，建议继续持有。”
    2. 信息源评估 (Recommender Reliability Score - RRS): 针对荐股人进行打分。
        * 历史平均回报率: 该推荐人所有推荐股票的平均收益率。
        * 胜率 (Hit Rate): 推荐的股票中，达到预设盈利目标（如盈利超过10%）的比例。
        * 最大回撤控制: 历史上其推荐的票出现的最大亏损幅度。
        * 推荐频率与周期: 是“快枪手”还是“长线价值派”？
        * 时间衰减加权: 近期的推荐表现权重应高于远期。
        * 输出: 每个荐股人会有一个0-100的可靠性评分，并附带标签，如“高胜率短线选手”、“价值投资挖掘机”、“风险厌恶型”。
* 功能:
    * Dashboard: 核心指标概览，如股票池总收益、最佳表现股票、最可靠荐股人等。
    * 股票池管理: 以列表形式展示所有推荐过的股票，可按荐股人、时间、评分等筛选排序。点击可查看详情，包括K线图、每次的评估记录和逻辑分析。
    * 荐股人管理: 展示所有荐股人的可靠性评分、历史推荐记录和成功案例。
    * 人工操作接口:
        * 手动录入: 补充非微信来源的推荐。
        * 信息修正: 修改NLP引擎提取错误的逻辑或股票代码。
        * 添加备注/标签: 为某个推荐或荐股人添加人工的观察和注释。
这是团队日常使用的核心工具，部署在内部通讯软件中（如企业微信、钉钉，甚至可以回连到微信）。
* 交互方式: 通过发送指令进行交互。
* 核心指令集:
    * /status [股票代码/名称] - 查询某只股票的最新状态、评分和分析。
    * /who [荐股人昵称] - 查询某位荐股人的可靠性评分和历史战绩。
    * /top [n] - 列出当前评分最高的n只股票。
    * /worst [n] - 列出当前评分最低/风险最高的n只股票。
    * /add [股票代码] [推荐逻辑] by [荐股人] - 快速手动录入一条推荐。
    * /alert on [股票代码] - 订阅某只股票的异动（如评分大幅变化、价格突破等）。
* 主动推送:
    * 每日战报: 每天收盘后，主动推送一份总结报告，包含股票池整体盈亏、评分最高的股票、重要逻辑验证事件等。
    * 实时告警: 当某只股票评分发生突变，或触发了人工设置的预警线时，立即推送消息。

5. 扩展性设计 (Extensibility)
为了应对未来可能的需求，系统在设计之初就预留了以下扩展接口：
1. 多信息源接入: 数据采集层设计为插件式。未来可以轻松编写新的Connector来接入钉钉、Telegram、邮件、甚至是券商研报PDF。
2. 可插拔评估模型: 评估引擎的核心算法应与主程序解耦。未来可以引入更复杂的因子模型、机器学习预测模型，作为新的评估策略，甚至可以进行A/B测试，对比不同模型的效果。
3. 策略回测模块: 基于已有的历史推荐数据和行情数据，可以开发一个回测模块，用于验证新的交易策略或评估模型的有效性。
4. API优先 (API-First): 所有核心功能（数据录入、查询、评估）都通过内部API实现。Web后台和Agent都只是这个API的客户端。这使得未来与其他系统（如交易系统）的集成变得非常容易。
5. 自定义动作/触发器: 用户可以自定义规则，例如“当荐股人A推荐股票，且其可靠性评分大于80时，自动发送一条高优先级通知到决策群”。

6. 技术栈建议 (Technology Stack)
* 后端: Python (生态完善，有大量数据分析、NLP和金融量化库)
    * Web框架: FastAPI / Django
    * NLP: spaCy / HanLP, Scikit-learn
    * 金融数据: Tushare / Baostock
    * WeChat交互: Wechaty (Node.js服务，通过RPC与Python后端通信)
* 前端: Vue.js / React (构建现代化的Web Dashboard)
* 数据库: PostgreSQL
* 任务调度: Celery + Redis (处理异步任务和定时评估)
* 部署: Docker + Docker Compose (简化部署和环境管理)
